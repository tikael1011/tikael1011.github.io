IndoorNav

Tech Involved: Java, Python, Weka, Android Studio, MATLAB.
libraries/Package in Python: scipy, numpy, matplotlib, statsmodels.
Porject Video: here. Great thanks to my teammate SonNam Nguyen.
This research project simulates the following (similar) situation: A gunman shows up inside a building, and people start to evacuate. An individual can get route to the nearest safe exit based on his location, gunman's location/movement and flow control from our mobile application.
Key highlights in my work:
♣ Developed android applications and debugged problems such as crashes and concurrency. Our client application is able to:
∆ start active scan, which helps Kismet detection and collect AP info.
∆ establish SSH connection (was FTP but for security reason we switched), upload scan results to server and download evacuation route from server.
∆ navigate user to exit by showing the latest route.
♣ Implemented new algorithm for wi-fi signal strength based indoor localization. There are many existing technologies/methods. But they either require extra hardware(e.g. Chrono MIT,2016) or can't meet our accuracy requirement(e.g. decision tree), or require relative long computation time. Thus a new model is required.
♣ Devised A* based evacuation route algorithm and reduced computation time significantly. As is known to all that A* has O(bd) runtime complexcity and O(bd) space complexcity in worst case. I dynamically changed the heuristics and setup a "stop searching" threshold, which avoids unnecessary computation.
♣ Improved localization accuracy at hallway intersections by utilizing image processing results. I utilized the detection to eliminate non-occupied location candidates, reduce computation time.

Evlucation:
In my opnion, this is most valuable part of this project. Because I learned a lot from this process.
Future work:
• Better UI design, multi layouts.
• Introduce time series method, may be able to improve accuracy.
• Image processing can also be used to achieve flow control, which might be able to evacuate people even faster.

Text Simplification

Tech Involved: Java, Python
libraries/Package in Python: NLTK
This is a co-op project with UMMS, designed for people with CD(cognitive disability). We want to replace complicate lexical words with simple synonyms while keep the gist. And so far our work is word level. 'Simplify' is different from 'summarize'. In short, summaried text is not necessarily simple.
Key highlights in my work:
♣ Utilized word2vec and then implemented other algorithms to find synonymic word candidates. The two main algorithms I implentmented were comparing cosø value and comparing euclidean distance.
♣ After find out candidates, I used different ranking method to get final result. For example, word "aaaaa" has the following candidates: "bbbb" (shortest length), "ccccc" (appeared in the orginal txt), "aabbaa" (highest ranking in dictionary, I tried both NLTK WordNet and own dictionary, which was generated from original text).

Future work:
It is hard to evaluate and what is the extact defination of 'simplier'. We tried manual rating, but it takes too much time and without large amount of experiments, the result is not convincible and can be subjective.
