IndoorNav

Main language: Java, Python
My work is mainly on the follwing parts:
• Develop user app. This app is able to
∆ start active scan AP, which helps Kismet detection and collect AP info
∆ establish FTP connection to server, send current AP info and receive evacuation route
∆ navigate user to exit based on preintalled map
• Wifi signal strength based localization. There are many existing technologies/methods. But they either require extra hardware(e.g. Chrono MIT,2016) or can't meet our accuracy requirement(e.g. decision tree, SVM), or require relative long computation time. Thus a new model is required.
• Combine image processing (Raspberry Pi + camera) to improve accuracy and reduce computation time, especially user is near an intersection. Through human recognition, we can also achieve flow control.
• Evacuation route algorithm design. Our current algorithm blocks all the hallways and rooms, which are directly connected to gunman's location and a cirle with center point of gunman's location and a radius of 20 meters.

Future work:
• Better UI design, multi layouts and SSH connection instead of FTP.
• Introduce time series method, may be able to improve accuracy.
• Current time complexity is O(nˆ2), n is number of points that describe the floor map. Try to optimize to O(nlog(n)) and calculate multi routes simultaneously.
